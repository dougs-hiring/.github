## 📌 Instructions
Merci d'utiliser cette grille d'évaluation afin de récupérer des critères objectifs pour comparer les candidats entre eux.

Cocher toutes les affirmations appropriées ci dessous.

---

## 😡 No Gos
- [ ] Aucun test ❌
- [ ] Logique dans les contrôleurs ❌
- [ ] Paramètres non typés (`any` ou `unknown`) ❌
- [ ] Pas de validation des inputs ❌
- [ ] Ne compile pas ❌
- [ ] N'est pas fonctionnel (validation des soldes erronée) ❌
- [ ] Pas d’isolation (tout est public) ❌

---

## 🤩 Should have
- [ ] Pas de code mort ou inutile
- [ ] Tests à l’interface métier
- [ ] Pas de tests inutiles
- [ ] README expliquant les hypothèses métier
- [ ] Pas d’erreurs de linter
- [ ] Complexité algorithmique non quadratique (O(n^a))
- [ ] Naming consistant et cohérent
- [ ] Mode strict
- [ ] Structure par couche
- [ ] Code facile à lire

---

## 👍 Nice to have
- [ ] Utilisation appropriée des fonctionnalités de Nest.js
- [ ] Complexité algorithmique ≤ 2(n + n*log(n))
- [ ] Utilisation d’objets riches / DDD

- [ ] Swagger
- [ ] Indique des pistes d’améliorations
- [ ] CI/CD
